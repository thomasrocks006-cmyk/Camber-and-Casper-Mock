Call System Architecture — Technical Handoff for Demo Implementation
Document Version: 1.0 Last Updated: April 22, 2026 Status: Production-Ready Reference Purpose: Complete technical specification for implementing voice call functionality in WorkshopOS demo

Executive Summary
This document provides everything needed to implement the Ironbark voice call system into a WorkshopOS mock demo application. The system is production-tested with 21+ successful test calls over April 20-22, 2026.

Core Stack:

Vapi.ai — Voice AI platform (conversation execution)
Twilio — Phone infrastructure (Australian numbers, SIP trunking)
Soldier Agent — Orchestration layer (strategy, monitoring, logging)
Campaign Strategy Director — Call logic engine (what to say, when, why)
Current Status: ✅ Working end-to-end with real voice calls, transcripts, recordings, and quality monitoring

System Architecture Overview
High-Level Flow

Vibe Check Engine (Real-Time Quality Scoring)
File: IronbarkRepo/agents/soldier/vibe_check.py

Scoring: 0-100 based on conversation signals

90-100: 🔥 HOT — Strong interest, asking questions
70-89: ☀️ WARM — Engaged, listening actively
50-69: 🌤️ COOL — Neutral, polite but reserved
30-49: ❄️ COLD — Disengaged, short answers
0-29: 🧊 FROZEN — Hostile, trying to end call
Signal Detection:

✅ Positive: Questions, agreement, timeline discussion
❌ Negative: Interruptions, objections, time pressure

FastAPI Backend
# demo_api.py
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
from pydantic import BaseModel
from soldier_agent import SoldierAgent
import os

app = FastAPI()
soldier = SoldierAgent(config={
    "vapi_api_key": os.getenv("VAPI_API_KEY"),
    "twilio_phone_number": os.getenv("TWILIO_PHONE_NUMBER")
})

class CallRequest(BaseModel):
    phone_number: str
    company_name: str

@app.post("/api/make-call")
async def make_call(request: CallRequest):
    lead = {
        "id": f"demo_{request.phone_number}",
        "company_name": request.company_name,
        "phone_number": request.phone_number,
        "industry": "workshop"
    }

    result = soldier.make_call(lead=lead)
    return result

# Serve HTML
app.mount("/", StaticFiles(directory=".", html=True), name="static")

Known Limitations
What Works ✅

Voice synthesis with Australian accent
Real-time transcription
Call recording and storage
Outcome classification

Webhook event processing
What Doesn't Work Yet ❌

Real-time prompt injection — Can't update strategy mid-call
Live intervention — Can't manually take over a call
Inbound calls — Only outbound currently configured
Vapi Limitations
Calls run on Vapi infrastructure (not your servers)
Configuration is "fire-and-forget" (can't modify mid-call)
Function calling is AI-initiated, not developer-initiated
Webhooks are one-way (Vapi → you, not you → Vapi)
